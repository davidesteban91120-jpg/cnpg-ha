# Security policy

## Supported versions

`cnpg-ha` is pre-1.0 software. Only the latest tagged release receives security
updates. We strongly recommend running the most recent version at all times.

| Version | Supported |
|---------|-----------|
| `main`  | ✅        |
| latest tagged release | ✅ |
| anything older | ❌  |

## Reporting a vulnerability

**Please do not open public GitHub issues for security vulnerabilities.**

Report privately to **david.esteban91120@gmail.com** with:

- A description of the vulnerability and its potential impact.
- Reproduction steps or a proof-of-concept.
- Affected versions / commit SHA if known.
- Your name and affiliation (optional — we are happy to credit you).

We will acknowledge receipt within **5 business days**, share an initial
assessment within **15 business days**, and aim to release a fix within
**90 days** for critical issues. We coordinate disclosure via CVE when
appropriate.

## Supply-chain guarantees

Every release artifact published from this repository ships with the following
attestations. See [`docs/SUPPLY_CHAIN.md`](docs/SUPPLY_CHAIN.md) for the full
verification workflow.

- **Container image** — `ghcr.io/davidesteban/cnpg-ha:<tag>` is built by a
  GitHub-hosted runner via the `image-build-sign.yml` workflow.
- **Keyless signature** (Sigstore / Cosign / Fulcio / Rekor) — every published
  image digest is signed with the workload identity of the GitHub Actions
  runner and recorded in the public Rekor transparency log.
- **SBOM** — SPDX 2.3 JSON, generated by Syft, attached to the image as a
  Cosign attestation (`spdxjson` predicate).
- **SLSA Level 3 provenance** — produced by the
  [`slsa-framework/slsa-github-generator`](https://github.com/slsa-framework/slsa-github-generator)
  reusable workflow and attached to the image as an `slsaprovenance` attestation.
- **Vulnerability scan** — every PR and tag runs `govulncheck`, `gosec`,
  `osv-scanner`, and `trivy image`. Builds fail on HIGH or CRITICAL findings
  (suppress with explicit `.trivyignore` entries reviewed in PR).
- **Secret scan** — `gitleaks` runs on every PR and as a pre-commit hook.
- **Dockerfile lint** — `hadolint` runs in CI and as a pre-commit hook.
- **OSSF Scorecard** — weekly snapshot at
  [securityscorecards.dev](https://securityscorecards.dev/viewer/?uri=github.com/davidesteban/cnpg-ha).

## Verifying a release locally

```bash
IMAGE=ghcr.io/davidesteban/cnpg-ha:v0.1.0
REPO=github.com/davidesteban/cnpg-ha

# 1. Verify the Cosign keyless signature.
cosign verify "$IMAGE" \
  --certificate-identity-regexp "^https://github.com/${REPO}/" \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com

# 2. Fetch and inspect the SBOM.
cosign download attestation --predicate-type=https://spdx.dev/Document "$IMAGE" \
  | jq -r '.payload | @base64d | fromjson | .predicate' > sbom.spdx.json

# 3. Verify SLSA L3 provenance.
cosign verify-attestation "$IMAGE" \
  --type slsaprovenance \
  --certificate-identity-regexp "^https://github.com/slsa-framework/slsa-github-generator" \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com
```

## Hardening at deploy time

Operators that want to **enforce** signature/provenance before admitting the
image into a cluster should pair the chart with a policy engine — e.g.
[Sigstore policy-controller](https://docs.sigstore.dev/policy-controller/overview/)
or [Kyverno](https://kyverno.io/policies/?policytypes=verifyImages%2520-%2520Cosign).
A sample Kyverno policy is provided in [`docs/SUPPLY_CHAIN.md`](docs/SUPPLY_CHAIN.md).
