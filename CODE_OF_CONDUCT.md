# Code of Conduct

The cnpg-ha project adopts the [Contributor Covenant, version 2.1][cc21]
as its Code of Conduct. By participating in this project — including
contributing code, opening issues, commenting on pull requests, or
taking part in Discussions — you agree to abide by its terms.

The full text is the canonical reference and is **incorporated by
reference** into this file:
<https://www.contributor-covenant.org/version/2/1/code_of_conduct/>

A short, plain-language summary:

- We want participation in this project to be a welcoming experience
  for everyone, regardless of background or experience level.
- Be respectful, constructive and patient with other contributors.
- Disagreement is fine; personal attacks are not. Critique the idea,
  not the person.
- Assume good faith and offer help rather than blame.
- Maintainers will moderate Covenant violations following the
  enforcement guidelines linked above.

## Scope

This Code of Conduct applies to:

- All cnpg-ha repositories and the GitHub organisation hosting them.
- Discussions, issues, pull requests and review comments.
- Any project communication channel (mailing lists, chat, conferences)
  operated by the maintainers.
- Public spaces when an individual is officially representing the
  project.

acting on the report. A report that involves a maintainer is reviewed
by the other maintainers, never by the subject of the report.

For security vulnerabilities, follow the separate private process
documented in [`SECURITY.md`](SECURITY.md).

## Enforcement

Enforcement follows the four-tier process defined in the Contributor
Covenant 2.1 (Correction, Warning, Temporary Ban, Permanent Ban). The
canonical wording of the enforcement guidelines lives at the upstream
URL above.
